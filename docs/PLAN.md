# 原子間相互作用の概念再現シミュレータ

## 1. 開発の背景と目的

本プロジェクトは、レナード・ジョーンズ（LJ）ポテンシャルが示す「近距離で猛反発、中距離で微弱な引力」という独特な挙動の背景にある物理ドラマを、視覚的・直感的に理解するためのシミュレータです。

既定の数式（$r^{-12} - r^{-6}$）をなぞるのではなく、**「原子核のプラスの力場（陽のバリア）」と「電子の自由行動圏（縄張り）」のせめぎ合い**を簡易的なルールでモデル化し、結果としてLJポテンシャルと同様の挙動が画面上に創発されることを目指します。

## 2. 技術スタック・構成

* **言語・環境：** Rust 100%
* **GUI / 描画フレームワーク：** `egui` + `eframe` (WebAssembly ターゲット)
* **公開方法：** `trunk` 等を用いて Wasm ビルドし、GitHub Pages にて静的配信

`egui` の `SidePanel` でパラメータ調整用のUI（スライダー等）を構築し、`CentralPanel` の `Painter` を使って粒子群を Canvas 描画します。

## 3. シミュレーションのモデル化方針（コアロジック）

各原子は、単なる点ではなく、以下の内部ステートを持つ構造体（`struct Atom`）として定義します。

### 3.1 原子の内部構造（Rust 構造体デザイン）

```rust
struct Atom {
    pos: egui::Pos2,       // 現在の位置 (x, y)
    vel: egui::Vec2,       // 現在の速度 (vx, vy)
    electron_offset: egui::Vec2, // 電子の現在地（中心からの偏りベクトル）
    // 必要に応じて、質量や個別の特性パラメータ
}

```

* **陽のバリア：** 中心から滲み出ているプラスの力場（半径 $R_{shield}$）。
* **電子の自由行動圏（縄張り）：** 電子が動けるエリア（半径 $R_{zone}$）。
* **`electron_offset`：** 自由行動圏内における、その瞬間の「電子の重心の偏り」を表すマイナスのベクトル。

### 3.2 毎フレームの計算アルゴリズム（`update` ループ内）

`egui` の `update` 関数内で、毎フレーム（`ctx.request_repaint()` を要求）以下の 3 ステップを回します。

#### 【ステップ1：電子の気まぐれなゆらぎ（確率的ノイズ）】

* 各原子の `electron_offset` に対して、ランダムな微小変化（`rand` クレート等を使用）を加えます。
* ただし、自由行動圏（半径 $R_{zone}$）からはみ出さないよう、ベクトルの長さ（マニグチュード）が $R_{zone}$ を超えたらクランプ（制限）します。

#### 【ステップ2：相互作用の判定（2つの原子の距離 $r$ に応じた処理）】

全原子ペアの距離 $r$ を計算し、働く力を合算します。

##### ①【近距離】縄張りの衝突（斥力）

* **条件：** 2つの原子の「電子の自由行動圏（縄張り）」が重なり合った場合（$r < 2 \times R_{zone}$）。
* **挙動：** 量子力学的な拒絶（パウリの排他律）として、**極めて強力な反発力（斥力）** を発生させます。重なりが深いほど強くなるようにします。

##### ②【中距離】誘引空間での連鎖（引力）

* **条件：** 縄張りは重なっていないが、お互いの「陽のバリア」の範囲内に入っている場合。
* **挙動：** 原子Aの電子が偏ることで右側のプラスが露出（バリアが剥がれる）すると、それが隣の原子Bの電子（`electron_offset`）を引き寄せ、結果として [Aのプラス] と [Bのマイナス] が引き合うマイルドな引力を発生させます。

#### 【ステップ3：位置と速度の更新】

* 合算された力を元に、速度ベルレ法（Velocity Verlet）等を用いて `pos` と `vel` を更新します。
* 画面の端（境界）に達した場合は、完全弾性衝突（速度ベクトルの反転）で跳ね返らせます。

## 4. 画面描画（`egui::Painter` による可視化）

物理的な背景が視覚的に伝わるよう、`egui::Painter` の形状描画メソッドを活用します。

* **原子の描画：**
* `painter.circle_filled` を重ね掛けして原子を表現。
* **「陽のバリア」：** 半透明の大きな円（例：`Color32::from_rgba_unmultiplied(0, 100, 255, 30)`）。
* **「中心核」：** 中央の小さな赤いドット。
* **「電子の偏り」：** 中心の座標から `electron_offset` だけズレた位置に、光る小さな点、または小さな半透明の円を描画。

## 5. 実装時の注意点（別エージェントへの申し送り）

* **大爆発（NaN/Infinity）のガード：**
どれだけ斥力を強くしても、フレームの隙間に粒子同士が完全に重なる（$r=0$）リスクがあります。ゼロ除算で `NaN` になりシミュレータがフリーズするのを防ぐため、距離の最小閾値ガード（`r = r.max(0.1)` 等）を必ず入れてください。
* **計算量の最適化と早期リターン：**
「陽のバリア」が届かないほど遠い粒子ペアは、ステップ2の重い計算をする前に `continue` させてください。
* **`egui` の連続描画（Repaint）：**
物理シミュレーションを滑らかに動かすため、`update` 関数の最後で必ず `ctx.request_repaint();` を呼び出し、無限ループで描画が更新されるようにしてください。
